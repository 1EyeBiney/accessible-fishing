Architectural Note: Scatter Circle vs Wind Mitigation Separation
The 5-tap model has two independent quality axes that must remain orthogonal in the resolver. Conflating them is the single biggest risk here.
Axis Set By Lives In Output
Scatter Quality Tap 3 (Apex Arrow) — deviation from the rod's loaded apex window castTiming.js scatterRadius: number (tiles)
Wind Mitigation Quality Duration match between Tap 2 (Spacebar start) and Tap 4 (Spacebar end) vs. duration of Tap 1→Tap 3 castTiming.js windMitigation: 0..1 (1.0 = 80% wind reduction, 0 = full wind)
Splash Quality Tap 5 (Splashdown Arrow) castTiming.js splashTier: SILENT | NORMAL | LOUD
`castTiming.js` is a pure 5-state FSM (IDLE → BACKSWING → MITIGATION_OPEN → APEX → MITIGATION_CLOSE → SPLASH → DONE). It records three timestamps from INPUT_* events, computes the three independent quality scalars, and emits one CAST_TIMING_RESULT payload. It never touches wind, grid, or boat state. If the player times out between any two taps, it emits CAST_WHIFF { reason: &#39;BIRDS_NEST&#39; } and tells inputAdapter to engage a 10–15s lockout — the real-time clock keeps ticking; that's the penalty (per your ruling #3).
`castResolver.js` consumes CAST_TIMING_RESULT plus { targetOffset, windVector, lureMaxRadius } snapshotted at Tap 1 (anchoring rule, D-011) and resolves landing in this declared order:
```
1. baseLanding   = targetOffset
2. scatterOffset = rng.stream('cast').inCircle(scatterRadius)   // Tap 3 quality
3. windPush      = windVector × flightTicks × (1 - windMitigation)  // Tap 2/4 quality
4. rawLanding    = baseLanding + scatterOffset + windPush
5. clampedLanding = clampToSpoolBoundary(rawLanding, lureMaxRadius)  // D-015 invisible wall
6. validateLegal(clampedLanding)  // land = abort + audio cue
7. emit CAST_LANDED { offset: clampedLanding, splashTier }
8. castSpookModel.applySplash(clampedLanding, splashTier)        // radial only (D-014)
```
The two axes never touch. Scatter is a vector sampled from a circle; mitigation is a scalar that scales the wind vector. They are summed in step 4 — that's the only place they meet, and it's a single line.
