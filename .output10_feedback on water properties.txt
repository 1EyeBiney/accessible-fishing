Data Modeling — Phase 1 of 3: Water Tile & Structure Index
NO CODE LOCK acknowledged. Schemas, tables, and conceptual JSON only — no executable JavaScript.
1. Tile Schema — Design Goals
Before the schema itself, the constraints driving every field choice:
Goal Reflected In
Sparse-Map storage (D-009) — only water tiles exist No isWater field; existence in the Map ≡ water
Boat draft filtering (D-022) depth.minM + depth.maxM
Cast resolver depth check & lure-vs-fish-depth match (D-032) depth.bottomM + cover.canopyDepthM
Spook decay over the continuous clock (D-013, D-036, H-012) state.spookLevel + state.spookLastUpdatedAtMs, computed-on-read
Micro-drift current vectors (Phase 2b) flow
Structure Index aggregation (D-005, H-002) tags[] — pre-classified, AI never re-derives
Replay determinism (H-004) All stochastic content seeded at generation; no runtime randomness in tile state
Single-purpose modularity Static traits vs. mutable state are partitioned at the top level
A tile is partitioned into three top-level branches:
`id` / `coord` — identity (immutable)
`traits` — physical, immutable after lake generation
`state` — mutable during a tournament; reset on TOURNAMENT_RESULTS
This boundary is critical: serialization for replays only needs state; world generation only writes traits; AI structure index only reads traits.tags plus selective live state queries.
2. Tile Schema — Conceptual JSON
```
{
  // --- Identity (immutable) ---------------------------------------------
  "id": "T:0341:0127",                  // stable string id; coord-derived
  "coord": { "x": 341, "y": 127 },      // lake-frame integer coords (D-009)
  // --- Physical traits (immutable post-generation) ----------------------
  "traits": {
    // Depth profile in meters. bottomM is the floor; minM/maxM are the
    // navigable water-column window for boats (after accounting for cover
    // and surface chop). poiGraph edges read minM to filter routes per D-022.
    "depth": {
      "bottomM":  4.20,                 // depth to lake bed
      "minM":     0.80,                 // shallowest passable water column
      "maxM":     4.20,                 // typically == bottomM unless overhang
      "slopeDeg": 8                     // local gradient; informs drop-off tags
    },
    // Bottom composition. Drives strike preference (species.js) and lure
    // snag probability (durability.js). Single dominant + optional secondary.
    "bottom": {
      "primary":   "GRAVEL",            // MUD | SAND | GRAVEL | ROCK | CLAY | SILT
      "secondary": "ROCK",              // optional; null if homogeneous
      "hardness":  0.65                 // 0..1, hooks/snag math
    },
    // Cover overlay. Cover is what occupies the water column above the
    // bottom. canopyDepthM = depth at which cover thins; lures shallower
    // than canopyDepthM may snag with probability 'snagRisk'.
    "cover": {
      "type":          "WEEDBED",       // NONE | WEEDBED | TIMBER | LILYPADS
                                        // | DOCK | BRUSHPILE | ROCKPILE | OVERHANG
      "density":       0.70,            // 0..1
      "canopyDepthM":  1.20,            // top of cover from surface
      "snagRisk":      0.35,            // 0..1, consumed by castResolver/durability
      "shadeFactor":   0.55             // 0..1, feeds species mood-to-eat (D-032)
    },
    // Static current vector (m/s). Wind-driven surface drift is added at
    // runtime by navigation/wind.js and is NOT stored here.
    "flow": {
      "vector":    { "dx": 0.05, "dy": -0.10 },
      "magnitude": 0.111                // precomputed for fast comparison
    },
    // Pre-classified structural tags. THIS is what structureIndex.js reads.
    // Tags are SET ONCE at lake generation by lakeGenerator.js using a
    // deterministic classifier over the surrounding tile neighborhood.
    // AI never re-derives these (H-002).
    "tags": [
      "DROP_OFF_EDGE",                  // tile sits on a >X°/m gradient
      "WEEDBED_INNER",                  // density > threshold + neighbors
      "AMBUSH_POINT",                   // composite: cover + depth break
      "SHADED_DAY"                      // shade > threshold during day window
    ],
    // Reachability classification at generation time. Used by the lake
    // generator's accessibility guarantee (early reachable hot features)
    // and by poiGraph edge construction.
    "reach": {
      "fromDockMin":  4,                // approx travel-cost class (1..10)
      "draftClass":   "ANY"             // ANY | SHALLOW | MID | DEEP_ONLY
    }
  },
  // --- Mutable state (cleared on TOURNAMENT_RESULTS) --------------------
  "state": {
    // Spook integer + last-update timestamp.
    // Decay is COMPUTED ON READ, not on tick:
    //   currentSpook = max(0, level - floor((nowMs - updatedAtMs) / SPOOK_DECAY_MS_PER_LEVEL))
    // This avoids per-tile bus traffic and replay non-determinism.
    // Writes happen only on splashdown (castSpookModel.applySplash) and on
    // tile-disturbance events (motor pass, etc.). See D-016 (radial spook).
    "spook": {
      "level":         0,               // 0..MAX_SPOOK (e.g., 5)
      "updatedAtMs":   0,               // tournament clock at last write
      "sourceEventId": null             // for debugging / replay diff
    },
    // Per-tile fish presence is NOT stored here. Fish are stored in
    // fish/population.js as a sparse list keyed by species + location.
    // This avoids O(grid) iteration during AI structure-index evaluation.
    "occupancy": {
      "fishCount":  0,                  // mirror of population query (write-through cache)
      "fishCountStaleAtMs": 0           // invalidates write-through cache
    },
    // Disturbance log for diagnostics + replay diffing. Bounded ring buffer.
    "events": []                        // [{ type, atMs, ... }]
  }
}
```
Field-by-field rationale (selected)
Field Why it lives here, not elsewhere
traits.depth.minM poiGraph.js edges aggregate this from waypoint tiles; boats with shallowDraftMin &gt; minM cannot use the edge. Stored per tile so the graph rebuild on lake regen is a pure aggregation, not re-derivation.
traits.cover.canopyDepthM Lets castResolver.js answer "did the lure pierce cover?" without consulting lures.js for cover semantics — separation of concerns.
traits.tags[] Single source of structural truth. Modifying classification rules means changing lakeGenerator, not the AI or the resolver.
traits.flow Static current. Wind is dynamic and stored in navigation/wind.js — a global, not per-tile. Combining them is a microDrift job.
state.spook.level + updatedAtMs Per-tick decay would be O(spookedTiles) every fight tick. Compute-on-read is O(1) per query and replay-deterministic because it's a pure function of (level, updatedAtMs, queryAtMs).
state.occupancy (write-through) Cache to keep AI structure-index queries cheap; canonical truth still lives in population.js. Stale flag invalidates safely.
Spook decay — explicit data contract
```
MAX_SPOOK             = 5
SPOOK_DECAY_MS_PER_LEVEL = 12000   // 12s per level
readSpook(tile, atMs):
    if tile.state.spook.level == 0: return 0
    decayed = floor( (atMs - tile.state.spook.updatedAtMs) / SPOOK_DECAY_MS_PER_LEVEL )
    return max(0, tile.state.spook.level - decayed)
applySplash(tile, splashTier, atMs):
    inc = splashTier == 'LOUD' ? 3 : splashTier == 'NORMAL' ? 1 : 0
    current = readSpook(tile, atMs)
    tile.state.spook.level       = min(MAX_SPOOK, current + inc)
    tile.state.spook.updatedAtMs = atMs
```
This contract belongs to casting/castSpookModel.js (per H-003). The tile is just a typed bag.
What is intentionally NOT in the tile
Excluded Why
isWater Existence in the sparse Map encodes it (D-009)
wind Global, owned by world/wind.js (or navigation/wind.js per current §9)
fishHere[] Population is sparse-list, not tile-attached, to keep iteration cost off the grid
Tile-local RNG seed All RNG is via rng.stream(...) (D-009 / §5d)
AI scores / "value" Derived in structure index, not stored on tile (would create cache invalidation hell)
3. Structure Index — Aggregation Model
world/structureIndex.js is the only module the AI is permitted to read for spatial reasoning (D-005, H-002). The shift to POI fast-travel (D-001) means the index is POI-keyed, not tile-keyed: each POI summarizes the tiles in its local micro-frame plus a small ring of context tiles.
Build phase
Triggered ONCE at lake generation, after tiles are tagged.
Pure function: `buildIndex(grid, poiGraph) → StructureIndex`.
Deterministic (seeded by `rng.stream('world')`); replays reproduce identical indices.
Update phase
Tags are immutable. The index does NOT incrementally update on spook or fish movement.
Live spook + live fish presence are queried at AI evaluation time via small targeted lookups, not folded into the static index. This keeps update cost zero between tournaments.
Query phase
Scheduled-AI cooldowns (D-013) call `structureIndex.evaluatePoi(poiId, ctx)` which:
1. Reads the precomputed POI entry (O(1)).
2. Augments with live ctx: `{ wind, timeOfDay, recentSpookSummary }`.
3. Returns a ranked feature list.
AI never iterates tiles. Worst-case cost is O(POI_count) per cooldown, capped by the scheduler.

```
  "frameRadius": 4,                       // micro-drift bound (D-002 area)
  "type":        "WEEDBED",               // enum mirroring §10 POI types
  // --- Static draft / accessibility profile (from constituent tiles) -----
  "draft": {
    "minDepthM": 0.8,                     // min bottom depth in frame
    "maxDepthM": 6.4,                     // max bottom depth in frame
    "draftClassesAllowed": ["ANY", "SHALLOW", "MID"]

  // --- Cover summary -----------------------------------------------------
  "cover": {
    "dominantType":  "WEEDBED",
    "coverageRatio": 0.62,                // fraction of frame tiles with cover
    "avgDensity":    0.55,
    "shadeAvg":      0.48,                // for time-of-day strike weighting
    "snagAvg":       0.31
  },
  // --- Bottom summary ----------------------------------------------------
  "bottom": {
    "primaryDistribution": {
      "GRAVEL": 0.55, "ROCK": 0.30, "MUD": 0.15
    },
    "hardnessAvg": 0.62
  },
  // --- Structural feature scores (precomputed; AI's primary input) -------
  // Each score is 0..1, derived from tile.traits.tags and depth/cover stats.
  // primeDirective.js maps these to action preferences by time-of-day.
  "features": {
    "DROP_OFF":     0.81,

  "timeAffinity": {

  },
  // --- Tile manifest (back-pointers; NOT iterated by AI) -----------------
  // Used only by castResolver, fishFinder, and debugging. AI does not walk
  // this list.
  "tileIds": [ "T:0341:0127", "T:0341:0128", /* ... */ ],
  // --- Live-state hooks (read at query time, not stored) -----------------
  "liveQueries": {
    "spookSummary":  "structureIndex.spookSummary(poiId, atMs)",   // returns {avg,max,recentSplashAtMs}
    "fishEstimate":  "population.estimateForPoi(poiId)"            // returns {byTier, totalKg}

  // --- Provenance --------------------------------------------------------
  "generatedFrom": {
    "seed":         123456789,
    "generatorVer": "1.0",
    "tileCount":    37

```
Why this shape satisfies §5d / §5e / D-005 / H-002
**Determinism (§5d):** Index is a pure function of seed + grid. Replays match.
**Time authority (§5e):** Live spook is read on demand using the canonical clock, never cached.
**AI perf (D-005, H-002):** AI reads one POI entry per cooldown cycle. No grid scan. No tile iteration in the hot path.
**Single source of truth:** Tags live on the tile. The index aggregates, never re-classifies.
