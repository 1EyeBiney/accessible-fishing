2. Headless Success Equation (per Scheduled Cooldown Tick)
Owned by ai/tournamentScheduler.js. Runs ONLY on a cooldown timer (D-013) — never per-physics-tick. No physical simulation: the bot resolves a "did I catch something this tick" boolean and, if true, samples a fish weight.
Cooldown Cadence
```
botCooldownMs = baseTickMs(tier) * (1.5 - 0.5 * skill) * weatherTickMod(lake)
  baseTickMs(tier=1..5) = [180000, 150000, 120000, 90000, 60000]    // 3min..1min
  weatherTickMod = optimal:0.85, neutral:1.00, harsh:1.30
```
Higher tier + higher skill = more catch attempts per tournament. Harsh weather slows everyone.
P(Catch) Equation
$$
P_{\text{catch}} = \text{base}(tier) \cdot S(skill) \cdot G(gearTier) \cdot W(weather, lake) \cdot M_{\text{lure↔species}} \cdot M_{\text{lure↔poi}} \cdot R(pressure)
$$
Term Definition Range
base(tier) Event difficulty floor: [0.45, 0.40, 0.35, 0.30, 0.25] [0.25, 0.45]
S(skill) 0.5 + 0.5 * bot.skill [0.5, 1.0]
G(gearTier) Bot's aiGearTier vs event tier: tierSkillCurve[matched/under/over] [0.7, 1.1]
W(weather, lake) 1 + bot.weatherSensitivity * (lakeWeather.bias[species] - 0.5) ~[0.6, 1.4]
M_lure↔species lure.speciesAffinity[chosenSpecies] (D-044/D-048) [0.0, 1.5]
M_lure↔poi dot-product of lure profile vs POI structureIndex tags × bot.poiSelectionBias [0.0, 1.5]
R(pressure) 1 - (PRESSURE_STRIKE_PENALTY * tile.pressure / MAX_PRESSURE) (D-039) [0.4, 1.0]
Clamp final to [0.05, 0.95]. Roll rngStream(&#39;aiCatch:&#39; + botId).next() &lt; P_catch → catch event.
Weight Sampling on Catch Hit
$$
\text{weightKg} = \text{lognormal}(\mu, \sigma) \cdot K(skill) \cdot T(targetPref) \cdot \text{tierWeightCurve}(gearTier)
$$
`μ, σ` from `species.weight.distribution`
`K(skill)` = `1 + 0.4 * skill` (skilled bots roll heavier)
`T(targetPref)` = `1 + 0.5 * targetPreference.TROPHY - 0.2 * targetPreference.VOLUME`
`tierWeightCurve` = `[0.85, 0.92, 1.00, 1.08, 1.18]`
Trophy-band rolls (top 10% of species range) require an extra gate: rng &lt; (0.10 * (0.5 + targetPreference.TROPHY)) — Trophy Hunters hit trophy weights ~3× more often.
POI Selection / Movement (between ticks)
```
zeroBiteMinutes = elapsed since last successful catch at current POI
if (zeroBiteMinutes > (5 + bot.patience * 25)):
    pick newPoi from poiGraph weighted by (poiSelectionBias × structureIndexScore × (1 - poi.pressure/MAX_PRESSURE))
if (currentPoi.pressure >= 3 AND rng < bot.adaptability):
    rotate to next lure in bot.lureRotation
```
All movement/lure-rotation decisions are deterministic against the seeded RNG sub-stream &#39;aiBrain:&#39; + botId.

4. Tournament Logic Integration — Same Bot, Different Win Conditions
A TROPHY_HUNTER bot's behavior is gated by tournament.rules.winCondition. The bot strategy module reads this on tournament-enter and switches its scoring objective function without changing its personality stats.
Trophy Hunter — HEAVIEST_BAG (top 5 fish)
Bag-fill phase: accept any fish ≥ species mean weight; ignore sub-mean catches (don't even count toward bag).
Once 5 fish in bag: **only swap a bag-fish if new catch > smallest-current-fish + 0.3kg** (avoid useless swaps).
POI selection biases toward `AMBUSH_POINT` and `DROP_OFF_EDGE` (trophy structure).
Late-tournament (last 25% of duration): if behind on weight, P(catch) ramped 1.15× via "desperation mode" — a dynamic skill bonus, not a personality change.
Trophy Hunter — TOTAL_CATCH_COUNT
Bot recognizes `targetPreference.TROPHY` is **misaligned** with the rules → applies `objectiveMismatchPenalty = 0.85` to `S(skill)`.
Switches to faster lures from `lureRotation` (those marked with shorter `actionType` ladders).
Patience effectively halved: `effectivePatience = patience * 0.5`. Bill, normally 10min stays, hops every 5min.
Still rolls heavier weights than peers (skill stays high), but loses count battles to GRINDER archetype because volume ≠ his strength.
Trophy Hunter — BIGGEST_FISH
Pure single-fish optimizer. Bag size = 1.
Will **discard** a current "best fish" mentally only if a new catch beats it.
POI rotation aggressive: hops at every zero-bite cycle (patience × 0.6).
Weight-roll trophy gate boosted: `(0.10 + targetPreference.TROPHY * 0.10)` — Bill hits trophies ~17% of his catches in this format vs ~13% in HEAVIEST_BAG.
Cross-Format Behavior Summary Table
winCondition Bag behavior Patience mod Lure rotation bias Mismatch penalty
HEAVIEST_BAG Top-5 swap-or-keep logic 1.00× Trophy lures favored none
BIGGEST_FISH Single-best discard logic 0.60× Trophy lures only none
TOTAL_CATCH_COUNT All catches count 0.50× Fast lures favored 0.85× skill
A GRINDER bot inverts these mismatches — punished in BIGGEST_FISH, rewarded in TOTAL_CATCH_COUNT. This creates rock-paper-scissors archetype dynamics the player learns to read by tournament schedule (D-056 weekly rotation).
